## Extension Blueprint | 扩展蓝图

### 36. Prefix Caching and Chunked Prefill

- **定位：** 解决长前缀重复计算和 prefill 过重的问题。
- **建议对象：** 选一个带长 prompt 的推理流程，观察前缀命中与重算差异。
- **重点关注：** prefix cache 命中、chunk 切分、prefill 复用、cache 结构设计。
- **预期产出：** 前缀命中图、重算节省表、cache splitter 的最小实现。
- **前置来源：** [22](./22_vLLM_PagedAttention.ipynb)、[24](./24_SGLang_RadixAttention.ipynb)

### 37. Multi-Token Decoding

- **定位：** 解决单 token decode 太慢的问题。
- **建议对象：** 选一个草稿-验证式解码流程，观察多 token 生成的收益。
- **重点关注：** 草稿 token、验证 token、接受率、回退机制。
- **预期产出：** 多 token 解码的接受率统计、回退路径说明、最小 simulator。
- **前置来源：** [23](./23_Speculative_Decoding.ipynb)、[21](./21_Decoding_Strategies.ipynb)

### 38. Decode Scheduling

- **定位：** 解决 prefill / decode / batch 排布问题。
- **建议对象：** 选一个小型推理任务，观察不同调度规则的吞吐差异。
- **重点关注：** 调度优先级、prefill 与 decode 排布、batch 重组、等待时间。
- **预期产出：** 调度对比表、排队 / 等待时间记录、最小 scheduler。
- **前置来源：** [21](./21_Decoding_Strategies.ipynb)、[23](./23_Speculative_Decoding.ipynb)、[36](./36_Prefix_Caching_and_Chunked_Prefill.ipynb)、[37](./37_Multi_Token_Decoding.ipynb)

### 41. KV Cache Scheduling

- **定位：** 观察 KV Cache 的复用、驱逐和优先级分配，把缓存调度和解码调度之间的边界看清楚。
- **建议对象：** 选一个多请求推理场景，观察 cache 调度对吞吐和延迟的影响。
- **重点关注：** cache 复用、驱逐策略、优先级、调度开销。
- **预期产出：** cache 调度记录、复用率统计、最小 scheduler。
- **前置来源：** [36](./36_Prefix_Caching_and_Chunked_Prefill.ipynb)、[38](./38_Decode_Scheduling.ipynb)、[40](./40_FP8_and_KV_Cache_Quantization.ipynb)
